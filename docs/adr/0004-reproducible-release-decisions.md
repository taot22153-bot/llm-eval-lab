# Use versioned rules and immutable evidence snapshots for Release Decisions

Release Rules are immutable, versioned inputs, and each `pass`, `fail`, or `manual_review_required` Release Decision stores its reasons, metrics, and a fingerprint of deterministic, semantic, runtime, and Human Review evidence so an unchanged Evaluation Run and rule reproduce the same snapshot. Decisive blocking failures, important new regressions, failed Human Reviews, and exceeded runtime budgets take precedence over unresolved Human Review; otherwise missing or unresolved evidence requires review before aggregate correctness and safety thresholds can pass or fail the release. Human Review gates the Candidate at execution level but never rewrites the automatic rule-level quality and safety metrics.
